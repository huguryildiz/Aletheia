# Worked exemplar — `uwsn-ankc`

**This is the pack's single domain-carrying file.** Every Aletheia skill was either
harvested from, or stress-tested against, one real research repository: `uwsn-ankc`, a
computational-science project optimizing underwater acoustic sensor-network deployments.
Skills and templates refer to this file namelessly ("see the pack's `examples/` directory")
so that the portability gate — zero domain terms in `skills/` and `templates/` — holds by
construction.

## The exemplar in one paragraph

`uwsn-ankc` jointly optimizes sensor **deployment**, non-uniform **κ-connectivity** (per-node
count of disjoint routes to a base station), and demand-anchored reliable **coverage** for an
underwater seismic-monitoring network, as a mixed-integer program (MILP, Gurobi) over an
acoustic channel grounded in Bellhop ray-tracing (Marmara Sea bathymetry); a GNN warm-start
phase amortizes solves across scenario ensembles. One funded 12-month project, one PI + one
MSc student, targeting a Q1 journal paper and a national conference paper. The stakes that
shaped the discipline: silent numerical breakage or an unreproducible sweep directly corrupts
the paper.

## Writer-agent patterns (documented, deliberately not shipped)

The exemplar also runs two *writer* agents Aletheia intentionally does not ship, because
execution automation is project-specific workflow:

- **`yl-runner`** (runner): executes campaigns under a written task contract (script, grid,
  seeds, time limits, output dir, definition of done); records evidence; never edits `src/`,
  never changes canonical constants, never nudges a grid to make INFEASIBLE cells disappear;
  stops and reports anomalies. Preflight: gate green, tree clean, native arch.
- **`opus-implementer`** (implementer): mechanical implementation turns dispatched from an
  orchestrating session — the orchestrator writes the contract and reviews; the implementer
  never makes gate decisions, never authors ADRs, never touches canonical values.

If you build your own runner/implementer, copy those contracts: they are the no-source-
mutation principle applied to agents that *must* write.

## A real meta.json, generalized

The exemplar's `results/baselines_2026-07-05/meta.json` is the direct ancestor of
`templates/results-meta.schema.json`. Field mapping: `constants_sha256/constants_path` →
`config_sha256/config_path`; per-script `*_blob_sha1` → `scripts[].git_blob_sha1`;
`arm64_verified` → `environment.arch_native_verified`; its `campaign.*.expectation` fields
("baselines INFEASIBLE in all/most realizations — pre-registered branch; recorded verbatim")
→ the schema's top-level `expectation`. The pre-registration in that file fired and became
the project's primary finding — the strongest argument this pack knows for writing
expectations down *before* running.

## What stayed domain-locked

Skills that only make sense next to this exemplar's machinery (solution-pool validation
against a specific solver API, transmission-loss cache management, bathymetry ingestion)
were left in the exemplar repo and are *described* here rather than shipped. If a future
portability review demotes a shipped skill, it lands in this file the same way.
