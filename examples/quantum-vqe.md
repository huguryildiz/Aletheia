# Worked example — variational quantum algorithms (illustrative)

> A **fictional** project, included only to show how Aletheia's domain-free skills instantiate
> in a real computational workflow. Not a real repository.

## The project in one paragraph

A group estimates ground-state energies of small molecules with a variational quantum
eigensolver, run on noisy simulators and hardware backends. Every energy is a statistical
estimate from finite measurement shots, so the discipline is about separating a real convergence
from shot noise, backend drift, and a badly-initialized ansatz.

## How the skills instantiate

| Skill | In this project |
|---|---|
| `project-layout` | `src/` package; gitignored `results/`, `backends/` caches with tracked READMEs; problem instances in `instances/` |
| `correctness-gate` | Critical modules = Hamiltonian construction and the expectation-value estimator. Gate = exact diagonalization on small systems must match the noiseless simulator within tolerance |
| `canonical-params` | Shot count, ansatz depth, classical optimizer, and noise model are frozen; changed only in a sweep or with approval |
| `environment-lock` | Pinned SDK + `numpy`/`scipy` versions; the simulator/backend version recorded per run (a backend update is a code change) |
| `data-fingerprint` | SHA-256 of the problem instance (molecular geometry, Hamiltonian terms) and the noise-model config in `meta.json` |
| `evidence-convention` | Each experiment lands in `results/<name>_<date>/` with raw measurement counts + `meta.json`; counts kept in scratch are a dark run |
| `reproducibility-provenance` | RNG seed for shot sampling logged; the energy-convergence figure is regenerated by one command from the logged estimates |
| `numerical-determinism` | Shot noise is intrinsic, not a bug — pin the sampling seed for reproducibility and report the estimator's mean ± standard error |
| `statistical-reporting` | Many circuit repetitions per point; report the energy estimator with error bars; optimizer-convergence curves carry CIs, not just the last value |
| `negative-results-ledger` | Barren-plateau initializations and non-converging ansätze are recorded, so no one re-tries a known dead end |
| `external-positioning` | Any "advantage" framing is gated behind a classical baseline (exact or tensor-network) computed first |

## Signature discipline here

`correctness-gate` via a classical oracle. The one move that keeps VQE honest is exact
diagonalization on a system small enough to solve classically: if the quantum pipeline can't
reproduce the known answer there, no result on a larger system is trustworthy. The gate is the
oracle, not the eye.
